1.0 - Your Account - To Top

Clicking My Account takes you to your account profile page. Here you can delete your account, change your password, upload a profile image of yourself, and fill out a biography.

Passwords must be at least 8 characters long and a mix of characters, just as required when registering. Never share your password with anyone else and never re-use passwords between websites. Your password cannot be seen by admins or any other users.

Images must be less than 100Kb in size and either a .gif, .png. or .jpg/.jpeg. You can only have one user image at a time and uploading a new one will delete any existing image.

The Profile can optionally use basic HTML (see X.X - An HTML Primer), but any custom styling, javascript, or advanced HTML will be discarded.

Your photo and profile are viewable by anyone clicking on an article you have written or if you have been flagged as a trainer.

Articles

2.1 - Edit an Existing Article - To Top

Selecting this feature from the navigation menu takes you an index of all existing articles, ordered from newest to oldest. Clicking the Edit link next to an article will allow you to edit or delete that article.

Take care when deleting articles, as this action cannot be undone under normal circumstances.

The Date field must be in "YYYY-MM-DD HH:MM:SS" format, invalid dates will be rejected.

The Author must be selected from the list of valid authors and cannot be manually entered.

The Title, Tags, and Body fields are identical to 2.2 - Publish a New Article.

2.2 - Publish a New Article - To Top

Selecting this feature from the navigation menu takes you to the article writing page. Here you can write new articles that will automatically appear on the front page of the site. Note: articles written by non-admins are moved to a queue where they must be approved by a Site Admin before publication.

The Title should be kept brief but pertinent. Keep in mind the space restraints titles have when appearing on the front page, particularly when viewed on a phone. The maximum allowed characters in a title is 200.

Tags are optional, but make your article easier to find in relevant searches. Tags must be separated by commas to work as intended. Good tags are general words related to your article content. For instance, an article about an upcoming powerlifting meet might be tagged "powerlifting, events, meets".

The Article Body is the main text of the article. The editor will allow you to insert links, images, and other useful elements, and you can optionally apply a number of existing CSS classes by clicking the HTML editor button in the top left (see X.X - An HTML Primer). If the article is lengthy, it would be a good idea to write it ahead of time in an external text file and then copy it into the CMS, so that nothing is lost in case of a browser or server error.

Please keep in mind that this will appear on the front page of the site and represents the business. Review thoroughly before submitting, depending on your permissions you may not be able to edit the article after submission.

Files

3.1 - Add or Delete a Gallery - To Top

Selecting this feature from the navigation menu takes you to a list of current image galleries, with buttons for their deletion, as well as a form to add a new gallery.

Deleting a gallery cannot be undone and will also delete all images currently in that gallery.

Any galleries you add or delete will automatically have appropriate navigation entries created/deleted.

3.2 - Upload a File - To Top

Selecting this feature from the navigation menu takes you to a form allowing you to upload select file types, either to include in an article or a related calendar event or social media post, or to an image gallery.

Permitted file types are .pdf with a maximum size of 5Mb or .gif, .png, or .jpg/jpeg with a maximum size of 1mb. .docx or .rtf documents should be converted to .pdf for easy web viewing, and videos and other file types should be uploaded to other appropriate systems (e.g. the business Youtube account).

You must select the appropriate file type for the action you are trying to perform, and appropriate gallery if trying to upload an image to a gallery. The gallery page will automatically include your new image if succesful, including automatically generating a thumbnail. If you need to delete an image from a gallery, you'll need to contact the SysAdmin at current time.

Checking the "Overwrite existing files of the same name?" box will cause any file you upload to overwrite existing files of the same name and extension with no further warning or checks. Attempting to upload a file that already exists without this box checked will display an error and no changes will be made.

After a sucessful upload, a modal window will appear with the URL of your file for you to copy and paste into the intended article/post.

Pages

4.1 - Add a New Page - To Top

Selecting this feature from the navigation menu allows you to create a new static page. This requires an understanding of HTML (#X.X HTML Primer), and it would be useful to have an understanding of available css classes for more than a basic page.

The Name field is required, accepts plain text, and will be used for the page's URL. The Name should never include spaces or special characters other than underscores ('_').

The Description field is optional but can be useful to increase search engine visibility. It accepts plain text and should be 155 characters or less.

The Title field is required, accepts plain text, and will appear in the browser title bar or tab.

The Priority field describes the importance of a page relative to all others on the site. It is used to a small extent by search engines but it is not overly important. If you're unsure, leave it at the default 0.5.

The Content field is required and accepts HTML, PHP, and Javascript. This is the body of the page. It will be automatically be given a standard container wrapper and the standard header and footer will be appended. You should not use <html>, <head>, <body>, <main>, or any other core structural tags beyond what is needed to mark up the content.

4.2 - Edit an Existing Page - To Top

Selecting this feature from the navigation menu takes you to an index of existing static pages. These comprise most of the basic pages of the site, but some (for example, image galleries) are dynamically generated and cannot be edited here. Clicking the Edit link will take you to forms allowing you to edit or delete that page. The fields are the same as those described in Add a New Page

Take care when deleting pages, as this action cannot be undone under normal circumstances. Before deleting a page which is currently part of site navigation and has navigation children, you will need to remove it from navigation first (See #5.1).

Editing may or may not require knowledge of web languages, depending on what you are trying to accomplish. This is an unavoidable trade-off of not overly restricting what changes can be made as well as not greatly complicating future wider site changes.

Please contact an admin if you have any questions.

4.3 - Edit Trainers - To Top

Selecting this feature from the navigation menu takes you to a page displaying all current trainers, allowing you to edit them, as well as a button leading to a form to add a new trainer.

Trainers which are Not Visible still exist in the database but can't be seen on the site. This is useful if, for example, you want to add a new trainer's profile but don't want it visible until you have a photo to add for them as well.

4.4 - Update Meet Records - To Top

Selecting this feature from the navigation menu takes you to a set of forms allowing you to update meet records for all current categories.

You should only need to manually input the Name and Record, all other required fields should have dropdown selection menus of their valid options. You must add a meet in the separate form provided before it can be selected for records.

Only one record can exist for an individual combination of classifications. Any records you try to add will safely be ignored if the record is lower than or equal to an existing record.

Bulk updating of records via csv file is planned but not yet implemented.

Site

5.1 - Edit Site Navigation - To Top

"Site Navigation" refers to the navigation buttons that appears at the top of each page of the site and is used to access other pages within the site. A "top level entry" is a button that is always visible, a "child entry" is a button that is only visible after hovering over or clicking another, and a "parent entry" is a top-level entry that has children. E.g. in the original setup, "gallery" is a parent top-level entry that has children of "bodybuilding", "community", etc, while "links" is a top-level entry with no children.

Selecting this feature from the navigation menu takes you to an index allowing you to view and edit the current site navigation arrangement. First is a representation of the current setup using bulleted lists, which is merely there as a visual aid. Below that is a table containing all the current entries, as well as buttons allowing you to Edit or Delete those entries. Finally comes a form allowing you to add a new entry.

When deleting entries, you cannot delete a parent entry that still has children. Move or delete the children first.

When editing or adding an entry, "Name" refers to the internally used name of the page. This name is part of the url (e.g. /name for a static page or /gallery/name for a gallery) and is the same as the "name" field when creating a static page or gallery using the CMS.

"Route" refers to path to the page, not including the domain name, e.g. a page at 'http://jakkedhardcore.com/foobar' has a route of '/foobar'. Routes are always unique.

"Title" is what site visitors will see on the button or list item for this entry. It can contain spaces and should have appropriate capitalization. It should be descriptive but very short, excessive length may cause problems with site styling.

"Description" will be shown to non-mobile visitors hovering over a button or to visitors using site readers or other accessbility devices. It does not need to be overly long but should give a basic indication of what the page's content contains.

The "Top-Level Entry?" checkbox determines whether the entry you are adding or editing will be a top level entry. You cannot move what is currently a parent entry from the top level until all its children are moved or deleted.

If the entry is not a top-level entry, the "Parent" dropdown will be available, allowing you to select which parent the entry you are adding or editing will become the child of.

Warning: This feature has unavoiably limited functionality. This CMS feature is intended to allow you to make minor changes within the existing layout without requiring web design knowledge, it cannot be used, by itself, to totally revise the site navigation structure. Mobile navigation will successfully adapt to any and all changes made, but adding child entries to any parent entry other than the current first two, or making changes to the titles of those first two parent entries, or adding more than five top-level entries, will require editing site styles for non-mobile visitors.

5.2 - Edit a Widget - To Top

"Widgets" refers to all other small site features not complex enough to warrant a full management system. Selecting this feature from the navigation menu will take you to a navigable index of the management pages for all current widgets.

The current widgets are: Alert Banner, Header Quotes, and Shelter Animal Gallery.

Admin

6.1 - Manage All Site Queues - To Top

Selecting this feature from the navigation menu takes you to the queue management index. All features of the site which allow outside people to upload content are handled through a queue system to prevent abuse.

Clicking the Edit link next to any item takes you to a view where you may approve or reject the submission, and edit it if necessary before approval. Once you submit approval, that item will be moved to the site and be removed from the queue. Rejected items are deleted permanently.

6.2 - Manage CMS Users - To Top

Selecting this feature from the navigation menu takes you to a list of current users. Clicking the Trainer checkbox will flag or unflag that user from showing up on the Trainers page. Clicking the Edit link next to a user will allow you to delete that user or change their roles.

Checking a box next to a user role that includes the permissions of other roles will uncheck and disable those lesser roles.

X.X - An HTML Primer - To Top

HTML is a means of "marking up", or structuring, text, as well as adding links, images, and other useful elements. HTML uses "tags", which are designated by '<' and '>' symbols. Most tags come in pairs, an opening tag and a closing tag, surrounding the text they apply to and identical aside from a '/' at the start of the closing tag.

For instance, <p>this is a paragraph</p> while this is not in the paragraph.

Some tags have no closing tag, instead including the '/' at the end of the lone tag. These are called "empty tags"

For instance, <img /> is an image tag, which are empty tags.

Tags can have attributes, which are included in the opening tag in the format 'attribute="value"' and with a space between the tag type and any attributes. Most attributes are optional, but some tags have required attributes.

For instance, an <img /> tag must have a 'src' attribute to be valid, with the value of the src attribute being the URL of the image you want shown,
e.g. <img src="http://example.com/example.jpg" />

Many tags can be safely nested as long as the closing tags and opening tags do not occur out of order. The innermost tag must be closed first, then working outwards.

For instance, <h1> <h2> <h3> </h3> </h2> </h1> is fine, while <h1> <h2> <h3> </h2> </h1> </h3> breaks.

Many tags can work together when nested.

For instance, an <a> tag creates a clickable text link when placed around text, but it can also be placed around an <img> tag to create an image which is a clickable link,
e.g. <a href="http://example.com"><img src="http://example.com/example.jpg" /></a>

Some tags only work in conjunction with other nested tags.

For instance, a <ul> tag creates an unordered list, which does nothing until you populate it with <li> tags, which are list items,
e.g. <ul><li>Item One</li><li>Item Two</li> <li>Item Three</li></ul>

HTML describes the structure of a page, but how that structure is presented to the viewer is governed by a seperate language called CSS. Existing CSS can be applied to HTML tags using the 'class' attribute. This is not something you'll generally use if you are reading this, but it can help you understand the content and intent of a page when editing it. Multiple classes can be applied to a single tag using a single class attribute by putting a space between them.

For instance, <p class="foo">this is a paragraph of class foo</p> <p class="foo bar">this is a paragraph of both class foo and class bar</p>

Basic HTML Tags
  • <p></p> - Paragraph, closing a paragraph creates a line break and trailing space.
  • <h1-6></h1-6> - Heading, large text with surrounding space. <h1> is the largest and <h6> the smallest.
  • <strong></strong> - Strong text, makes the text slightly bolder and/or larger. Use sparingly for emphasis.
  • <em></em> - Italicized text. Use sparingly for emphasis.
  • <a href=""></a> - Link, href value must be the URL of the page or file you want the link to point to.
  • <img src="" /> - Image, src value must be the URL of the image you want shown.
  • <ul></ul> - Unordered (Bulleted) List, does nothing without list items.
  • <ol></ol> - Ordered (Numbered) List, does nothing without list items.
  • <li></li> - List Item, must be within an unordered or ordered list.
  • <br /> - Line break, should only be used to insert a line break in the middle of a paragraph, if you want a new paragraph with proper trailing space you should close the existing paragraph tag and open a new one.
